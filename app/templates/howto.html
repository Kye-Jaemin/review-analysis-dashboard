1. 이 앱이 하는 일

Google Play · Apple App Store · Reddit · 임의 웹사이트에서 사용자 리뷰를 모은 뒤, 자동으로 Top 10 카테고리를 추출하거나 본인이 정의한 수동 카테고리 트리로 분류하고 5단계 감성(매우 부정 ~ 매우 긍정)을 평가합니다. 결과는 조사 카드를 클릭해서 대시보드·리뷰·마인드맵에서 비교·필터링·내보내기 가능, 그리고 업체 강/약점 분석 페이지에서 강점/약점도 한눈에.

조사 카드(Investigation card)란?
분석을 실행할 때 "분석할 소스 + 분류 방식(자동 / 수동)"을 묶어 이름을 붙여 저장한 카드입니다. 대시보드와 리뷰 페이지 상단에 그리드 형태로 표시되며, 카드를 클릭하면 모든 패널이 그 카드의 범위로 즉시 좁혀집니다. 마인드맵·자동 카테고리도 카드별로 따로 저장됩니다.

2. 빠른 시작 (5단계)

  1. 소스 추가 — 리뷰 수집에서 분석할 앱·서브레딧·URL 등록
  2. 수집 실행 — 소스 페이지에서 각 소스의 수집 시작 클릭 (액세스 코드 필요)
  3. (선택) 수동 카테고리 정의 — 자동 분류로 시작해도 무방. 직접 정의한 분류 체계가 필요하면 수동 카테고리에서 트리 생성
  4. 분류 + 긍부정 평가 실행 — 리뷰 분류 및 긍부정 평가에서 소스 선택, 분류 모드 (자동 / 수동) 선택, 이름 입력 후 실행 (액세스 코드 필요)
  5. 결과 확인 — 대시보드 진입 후 카드 클릭으로 즉시 그 카드 범위 보기. 업체 강/약점 분석 페이지에서 업체별 강점/약점도 자동 집계

3. 리뷰 수집 — 4가지 채널 (사이드바: 리뷰 수집, 구 "소스")

3.1 Google Play

  1. 소스 → 소스 유형 = Google Play
  2. 검색창에 앱 이름 입력 (예: spotify)
  3. 국가(country) 선택 — 리뷰 언어에 영향
  4. 후보 카드(아이콘 + 개발자명 + 평점) 중 원하는 항목 클릭
  5. 별칭·최대 개수 조정 → 추가

3.2 Apple App Store

절차는 Google Play와 동일. 단 country가 더 중요 — Apple은 국가별 별도 리뷰 풀입니다.

Apple RSS 제한: Apple 공식 RSS 엔드포인트가 일부 (앱, 국가) 조합에서 빈 응답을 반환합니다 (예: MacroFactor 의 us 스토어). 수집이 0건이면 gb (영국) 또는 jp (일본) 등 다른 국가로 다시 추가해보세요. 콜렉터는 sortBy=mostRecent + mostHelpful 양쪽을 자동 시도해 최대한 긁어옵니다.

3.3 Reddit

사전 준비:

  1. reddit.com/prefs/apps에서 script 타입 앱 생성 (이메일 인증 안 됐으면 먼저 인증)
  2. 발급된 client_id + secret을 Render Environment에 등록:
    • REDDIT_CLIENT_ID
    • REDDIT_CLIENT_SECRET
    • REDDIT_USER_AGENT=web:review-collector:v0.1 (by /u/your_id)
  3. Render 재시작 후 소스 → Reddit 선택 → 서브레딧 검색 → 선택 → 추가
Reddit 수집은 느립니다: 기본값 500 submission × 코멘트 20개씩 + 레이트 리밋 대기 = 4–10분. 진행 바가 실시간 업데이트되도록 스트리밍 처리되어 있으니 멈춘 것처럼 보여도 정상입니다.

3.4 사용자 지정 URL

  1. 소스 → 소스 유형 = Custom URL
  2. URL 목록 입력 (한 줄에 하나)
  3. CSS selector 옵션 (비워두면 readability로 자동 본문 추출)
  4. JS 렌더 필요 시 Playwright 옵션 (Render 무료 플랜에선 메모리 제약으로 비활성화 권장)

4. 카테고리 트리 만들기 (수동 모드용)

자동 분류로 시작할 때는 이 단계 건너뛰기 가능. 본인이 정의한 분류 체계가 필요하면:

  1. 수동 카테고리 → 루트 카테고리 추가
  2. 이름과 설명 입력 — 설명이 LLM 분류 기준
  3. 각 노드의 하위 추가로 자식 생성
설명 작성 팁: "결제 관련 (구독, 환불, 가격)" 같이 1~2줄로 키워드를 적어두면 LLM 분류 정확도가 크게 올라갑니다. 빈 설명은 분류 품질 저하의 가장 큰 원인.

5. 리뷰 분류 및 긍부정 평가 — 자동 vs 수동 (사이드바: 리뷰 분류 및 긍부정 평가, 구 "분석")

분석 페이지의 가장 큰 선택지는 분류 방식입니다:

5.1 자동 모드 (기본 추천)

  • LLM 이 이 데이터셋에서 Top 10 주제를 직접 뽑아냅니다 — 카테고리 트리 미리 정의 불필요
  • 여기에 단순 긍정 / 단순 부정 2개 고정 버킷이 자동 추가 → 총 12개 카테고리
  • "great", "👍" 같은 짧은 일반 칭찬은 단순 긍정에 모이고, 진짜 주제 (UX 문제, 기능 누락 등) 는 Top 10 에 배치됨
  • 분석 후 5개 감성별 마인드맵도 자동 생성·저장

5.2 수동 모드

  • 본인이 정의한 카테고리 트리에서 분류 (위 4번 섹션)
  • 여러 개의 루트 카테고리 멀티 선택 가능 — 그 subtree 들만 LLM 에 노출
  • 같은 소스에 여러 수동 카드를 만들어도 각자 독립 — per-card 정션 테이블로 격리되어 다른 카드 결과 안 깨짐

5.3 공통 설정

  1. 모드: 미분석만 (보통) / 실패만 재시도 / 전체 재분석
  2. 분석 대상 소스: 체크박스 멀티 선택. 미체크 시 전체
  3. 모델:
    모델속도·비용정확도추천
    Haiku 4.5매우 빠름·저렴좋음대량 분류, 초기 탐색
    Sonnet 4.6균형매우 좋음일반 분석
    Opus 4.7느리고 비쌈최고중요한 인사이트
  4. 요약 언어: UI 언어가 기본 (한국어 UI → 카테고리명·리뷰 요약도 한국어). 영어 / 한국어 / 자동 선택 가능
  5. 분류 민감도(최소 confidence): LLM 의 confidence 가 이 값 이하면 카테고리는 미할당, sentiment 만 기록 → "강제로 끼워맞춘 분류" 줄임
  6. 유/무료 사용자 구분 (BETA): 체크하면 리뷰에서 paid/free/unknown 도 자동 판별. 대시보드 자동 카테고리 타일에 유료/무료 토글 등장
  7. 현재 조건에 해당하는 리뷰: 체크박스·모드 변경에 따라 분석 대상 건수가 실시간 표시
  8. 조사 카드로 저장:
    • 이름 칸이 자동으로 <선택한 소스들>_<선택한 카테고리들> 형태로 채워짐
    • 그대로 두거나 직접 수정. 비우면 카드는 만들지 않고 분석만 실행
  9. 분석 실행 → 진행률 표시 (2초 간격 폴링) → 완료시 confetti + 토스트 알림
액세스 코드: 분석 실행 버튼을 누르면 4자리 코드 입력창이 뜹니다. 같은 탭 안에서는 한 번만 입력하면 됩니다 (sessionStorage 기억). 코드 미보유시 jaemin.kye@samsung.com 으로 문의.

6. 조사 카드 관리

6.1 정렬 & 그룹핑

  • 카드는 자동으로 업체별 그룹으로 묶임 (display_name 기준 — BitePal AS + GP + Reddit 자동 한 묶음)
  • 같은 업체 안에서 자동 카드가 먼저, 수동 카드가 뒤
  • 드래그-앤-드롭으로 카드 순서 변경 — DB 에 영구 저장
  • 같은 업체에 카드 2개 이상이면 스택 상태로 압축 (피크 카드만 보이고 뒤에 카드 그림자) — 카드의 +N 더 보기 ▾ 칩 클릭하면 부채꼴로 펼쳐짐

6.2 카드 정보

  • 카드 좌상단: 업체 이름 (+ 생성 날짜 — 드래그 재정렬해도 안 바뀜)
  • 카드 우상단: 그 카드의 총 리뷰 수 (대용량 강조)
  • 카드 본문: 소스 행 (앱 아이콘 + 이름 + 플랫폼 아이콘 + 그 소스 리뷰 수)
  • 카드 하단: 자동/수동 카테고리 칩 + 액션 버튼들

6.3 카드 액션 (우하단)

  • 카드 내보내기 — 이 카드 하나만 JSON 으로 export (그 카드의 리뷰·분석·자동 카테고리·마인드맵 모두 포함, 다른 카드 분석은 영향 0)
  • 숨기기 / 복원 — 카드를 대시보드에서 숨김 (데이터는 보존). 헤더의 "👁 숨긴 카드 N개 보기" 토글로 다시 표시
  • 삭제 — 카드 + 자동 카테고리 + 정션 + 그 카드의 마인드맵 모두 CASCADE

6.4 카드 가져오기

  • 대시보드 / 리뷰 페이지 상단의 📥 카드 가져오기 로 JSON 업로드
  • 같은 앱 소스는 자동으로 dedup — 같은 앱의 BitePal 소스 가 또 생기지 않음
  • import 한 카드는 가시 상태로 추가 (숨김 플래그 무시)

7. 대시보드 패널

활성 카드에 따라 자동/수동 모드 패널이 다르게 표시됩니다:

7.1 공통 (모든 카드)

  • 요약 4 카드: 총 리뷰 / 평균 감성 점수 / 평균 별점 / 최근 수집
  • 소스별 도넛 — 플랫폼 아이콘 + 평균 별점 포함
  • 감성 원인 마인드맵 (다음 섹션)
  • 시간 추이 — 주간/월간 × 5단계 분포 / 평균 점수
  • 최근 리뷰: 카드 범위의 최신 10건

7.2 자동 모드 패널

  • 자동 카테고리 Top 12 도넛 — 정렬: 응답 많은 순. 가운데에 총 카운트
  • 오른쪽에 카테고리 범례 (이름 + 카운트 + %)
  • 하단에 progress-bar 타일 그리드 — 각 타일이 share-of-total 만큼 채워지는 fill 애니메이션, sentiment 5밴드 분포 미니바
  • 타일 클릭 시 그 카테고리의 리뷰 모달 (감성 필터 + 유료/무료 토글)
  • 리뷰 모달의 각 행에 🌐 번역 토글 — UI 언어로 즉시 번역 (1회만 LLM 호출, DOM 캐시)

7.3 수동 모드 패널

  • 루트 카테고리별 감성 도넛 + 범례
  • 카테고리별 도넛 + 우측 범례 (각 leaf 의 5밴드 미니바 + 카운트)
  • 활성 카드에 수동 분류 결과가 없으면 (즉, 자동 모드만 돌렸음): "수동 분류 결과가 없습니다" 안내 + 분석 페이지로 deep link (소스 + root + manual mode + mode=all 미리 채워짐)

7.4 인터랙션

  • 카드 hover → 3D tilt 효과 (perspective + rotateX/Y)
  • active 카드 → aurora glow (테두리 둘레 빛이 회전)
  • 도넛 빈 공간 클릭 → reveal 애니메이션 재생
  • 마인드맵 ⟳ 새로고침 버튼 → 활성 카드 기준 다시 로드 (LLM 비용 없음, 캐시 우선)

8. 업체 강/약점 분석 페이지 (사이드바: 업체 강/약점 분석, 구 "업체 분석")

업체 강/약점 분석 메뉴 — 모든 소스를 업체 stem (display_name 의 첫 단어) 기준으로 묶어 강점/약점을 집계합니다.

  • 신규 LLM 호출 0건 — 기존 분석 결과 가공만
  • 각 업체 카드:
    • 업체 이름 + 플랫폼 칩 (어느 스토어/Reddit 에서 데이터 수집했는지)
    • 총 리뷰 / 평균 감성 / 평균 별점
    • 5밴드 sentiment 분포 바
    • 💪 강점 Top 5 (긍정 비율 높은 자동 카테고리 순)
    • ⚠️ 약점 Top 5 (부정 비율 높은 자동 카테고리 순)
업체 alias 자동 병합: 같은 회사가 스토어마다 다른 이름으로 등록된 경우 자동으로 한 row 로 합쳐집니다:
  • Google Health (Fitbit) / Fitbit → 같은 업체
  • Weight Watchers Program / WeightWatchers / WW → 같은 업체
  • AppleFitnessPlus / Apple Fitness → 같은 업체
단순 긍정/부정 버킷은 강점/약점 ranking 에서 제외 — 이건 진짜 강/약점이 아니라 sentiment 덤프이기 때문.

8.1 강점·약점 클릭 → 세부 원인 분석 (NEW)

각 강점/약점 Top 5 항목을 클릭하면 모달이 열리고, LLM이 해당 카테고리 리뷰 최대 100건을 읽어 4–6개의 세부 원인을 추출해 가로 막대 차트로 보여줍니다.

  • 샘플 범위: 해당 (벤더, 카테고리, 감정 밴드) 조합의 리뷰. 강점은 긍정+매우긍정, 약점은 부정+매우부정 합산
  • 차트: 가로 막대 (원인별 리뷰 수). 강점은 녹색, 약점은 빨강
  • 예시 리뷰: 각 원인을 펼치면 대표 리뷰 2–3개 표시
  • 📌 저장: 모달 하단 💾 카드로 저장 → DB에 영구 저장. 다음 클릭부터 LLM 호출 0건으로 즉시 불러옴. Render 재배포로 메모리 캐시가 날아가도 살아남음
  • 저장된 항목은 강점/약점 텍스트 옆에 📌 아이콘으로 표시됨
  • 🔄 다시 분석: 명시적 opt-in. 새 리뷰가 쌓였을 때 카드를 갱신
  • 🏷️ 이름 변경 / ❌ 삭제: 저장된 카드만 적용 가능
  • LLM 결정성: temperature=0 으로 호출되어 같은 데이터엔 같은 결과 반환
  • truthful count: LLM이 estimate한 원인별 count를 서버측 deterministic 샘플 수로 cap → 차트와 실제 리뷰 수 일치 보장
예시 시나리오: Weight Watchers Program의 체중 감량 성과 공유(강점) 클릭 → 모달 열림 → "지속적 식단·습관 형성 (24)", "커뮤니티 응원 (18)", "포인트 시스템 (17)" 같은 원인이 차트로 표시 → 각 원인 펼치면 실제 리뷰 인용 → 💾 저장.

9. 경쟁력 분류 페이지 (사이드바: 경쟁력 분류(베타))

경쟁력 분류(베타) 메뉴 — 업체 강/약점 분석에서 내보낸 CSV를 업로드하고, 분류 기준 경쟁력 요소(예: "음식 Vision AI 정확도")를 입력하면 LLM이 CSV의 각 강점 row를 그 요소와의 매칭도로 점수 매겨 한 줄씩 분류해 보여줍니다.

전체 흐름:
  1. 업체 강/약점 분석 페이지에서 📥 CSV 내보내기 클릭. 저장된 원인 분석 포함 체크 권장 (세부 요인 열 자동 채워짐)
  2. 경쟁력 분류 페이지에서 그 CSV 업로드 (드래그앤드롭 또는 클릭)
  3. 경쟁력 요소 입력 + 임계값 조정 후 [분석]
  4. 매칭된 강점 row가 한 줄씩 테이블로 분류 표시

9.1 동작 방식

  • 매칭 방식: LLM(Claude, temperature=0)이 CSV의 distinct 카테고리들과 입력 요소의 의미 일치도를 0–1로 채점. 키워드가 정확히 일치하지 않아도 의미가 가까우면 매칭. 같은 입력은 항상 같은 점수
  • 대상: CSV의 type=strength row만 분류. 약점 row는 무시 (이번 분류 화면에서는 강점 비교에 집중)
  • 임계값: 기본 0.5. 슬라이더로 0.3~0.9 조정 가능. 미만은 결과에서 제외
  • 출력 컬럼: 매칭도 / 업체 / 강점 요소(카테고리) / 긍정 % / 응답수 / Wilson / 세부 요인(인과)
  • 세부 요인 열: CSV의 reasons 열을 그대로 표시. /vendors CSV 내보낼 때 "저장된 원인 분석 포함" 체크돼 있어야 채워짐
  • 정렬: 매칭도 내림차순, 동률은 긍정% 내림차순

9.2 카드로 저장

분석 결과 상단의 💾 카드로 저장 버튼 → DB에 CSV 스냅샷 + 결과 row가 함께 영구 저장:

  • 좌측 사이드바에서 저장된 카드 클릭 → LLM 호출 없이 즉시 같은 결과 표시. CSV 파일이 사라져도 카드만 있으면 재현 가능
  • 🔄 다시 분석: 저장된 같은 CSV로 LLM 재호출. 동일 입력 + temperature=0이라 결과 거의 같지만, 모델 변경 시 차이 확인 용도
  • 🏷️ 이름 변경 / ❌ 삭제: 인라인 rename + 영구 삭제
  • 사이드바 카드 행에 📄 N → 💪 M 형식으로 (CSV 행 수 → 매칭된 강점 수) 표시
예시: /vendors에서 14개 업체 강점 CSV (50행) 내보내기 → 경쟁력 요소 "음식 Vision AI 정확도" 입력 → 매칭 8건 (Cal AI / BitePal / SnapCalorie 등의 AI 인식 관련 강점) → 각 행에 매칭도 + 인과 원인 표시.

1. What this app does

Pull reviews from Google Play, Apple App Store, Reddit, and arbitrary web pages; either let the LLM auto-derive a Top 10 category set from the data or classify into your own manual category tree; grade each on a 5-band sentiment (very negative → very positive); then use Investigation cards to drill into any saved (sources × classification) subset. The Vendor analysis page then rolls up everything by vendor and surfaces strengths/weaknesses.

What's an Investigation card?
When you run analysis you bundle the chosen sources and classification mode (auto / manual + root scope) under a name. That bundle becomes a card on the dashboard and reviews pages. Click the card and every panel filters to that subset. Mind maps and auto-categories also live per-card.

2. Quick start (5 steps)

  1. Add a source — register apps, subreddits, or URLs in Sources
  2. Collect — click Start collection on each source card (access code required)
  3. (Optional) Define manual categories — skip this if you'll start with auto mode. Build a tree in Manual categories only if you need your own taxonomy
  4. Run analysis — in Analyze, pick sources, pick mode (auto/manual), name the card, run (access code required)
  5. Inspect — opening the Dashboard auto-activates a card. Visit Vendor analysis for cross-card strengths/weaknesses.

3. Add a source — 4 channels

3.1 Google Play

  1. Sources → source type = Google Play
  2. Search the app name (e.g. spotify)
  3. Pick a country — affects review language
  4. Click a candidate card → adjust label / max count → Add

3.2 Apple App Store

Same flow as Google Play. Country matters more here — Apple has a separate review pool per country store.

Apple RSS quirks: The public RSS endpoint returns empty for some (app, country) combos (e.g. MacroFactor in us). If collection comes back at 0, try gb or jp. The collector also tries both sortBy=mostRecent and mostHelpful to maximise coverage.

3.3 Reddit

  1. Create a script-type app at reddit.com/prefs/apps (verify email first if needed)
  2. Set Render env vars: REDDIT_CLIENT_ID, REDDIT_CLIENT_SECRET, REDDIT_USER_AGENT=web:review-collector:v0.1 (by /u/your_id)
  3. Sources → Reddit → search subreddit → pick → add
Reddit collection is slow: default 500 submissions × comments + rate limit waits = 4–10 minutes. The progress bar streams per-item via an asyncio.Queue, so it moves continuously — if it looks frozen it isn't.

3.4 Custom URL

  1. Sources → source type = Custom URL
  2. One URL per line
  3. Optional CSS selectors (empty = readability auto-extract)
  4. Playwright toggle for JS-rendered pages (off by default on Render free for memory reasons)

4. Build a manual category tree (optional)

Skip this if you'll use auto mode. Only needed if you want your own taxonomy:

  1. Manual categories → Add root category
  2. Enter name + description — the description is the LLM's classification rubric
  3. Use each node's Add child to grow the tree

5. Run analysis — auto vs manual

The biggest choice on the analyze page is the classification mode:

5.1 Auto mode (default, recommended)

  • The LLM extracts the Top 10 themes directly from this dataset — no pre-defined tree needed.
  • Two fixed buckets are added: Simple praise / Simple complaint for short generic reviews like "great" or "meh" → 12 categories total.
  • 5 sentiment mind maps are auto-generated and saved right after analysis.

5.2 Manual mode

  • Classify into your defined category tree (section 4)
  • Multi-select multiple root categories — only those subtrees are exposed to the LLM
  • Multiple manual cards over the same sources stay isolated via the per-card junction — re-running one card doesn't break the others.

5.3 Shared settings

  1. Mode: unanalyzed only (typical) / retry failed / re-analyze everything
  2. Source scope: multi-select checkboxes; unchecked = every source
  3. Model:
    ModelSpeed / CostAccuracyWhen
    Haiku 4.5Fast / cheapGoodBulk, exploration
    Sonnet 4.6BalancedVery goodGeneral use
    Opus 4.7Slower / pricierBestHigh-stakes insights
  4. Summary language: defaults to your UI language; English / Korean / Auto override available
  5. Classification sensitivity (min confidence): drops category assignments below the threshold (sentiment is still recorded)
  6. Separate paid/free users (BETA): tier inference from review content; surfaces a paid/free toggle on auto-cat tiles
  7. Live count: "Reviews matching the current selection" line updates as you toggle
  8. Save as investigation card:
    • Auto-filled with <sources>_<categories>
    • Keep / rewrite / empty (run without saving a card)
  9. Run analysis → polled progress, confetti + toast on completion
Access code: the analyze button prompts for a 4-digit code. One entry per browser tab (sessionStorage). Contact jaemin.kye@samsung.com if you need access.

  • Per-vendor card:

    • Total reviews / avg sentiment / avg rating
