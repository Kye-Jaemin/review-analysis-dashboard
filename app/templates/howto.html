1. 이 앱이 하는 일

Google Play · Apple App Store · Reddit · 임의 웹사이트에서 사용자 리뷰를 모은 뒤, 본인이 정의한 계층형 카테고리로 자동 분류하고 5단계 감성(매우 부정 ~ 매우 긍정)을 평가합니다. 결과는 조사 카드를 클릭해서 대시보드·리뷰·마인드맵에서 비교·필터링·내보내기 가능.

조사 카드(Investigation card)란? 분석을 실행할 때 "분석할 소스 + 분류 범위(루트 카테고리)"를 묶어 이름을 붙여 저장한 카드입니다. 대시보드와 리뷰 페이지 상단에 그리드 형태로 표시되며, 카드를 클릭하면 모든 패널이 그 카드의 범위로 즉시 좁혀집니다. 마인드맵도 카드별로 따로 저장됩니다.

2. 빠른 시작 (5단계)

  1. 소스 추가 — Sources에서 분석할 앱·서브레딧·URL 등록
  2. 카테고리 정의 — Categories에서 분류 기준 트리 생성
  3. 수집 실행 — Sources 페이지에서 각 소스의 수집 시작 클릭
  4. 조사 카드 생성 + 분석 — Analyze에서 분석할 소스와 카테고리를 체크하고 이름을 입력 후 실행 (이름은 자동으로 소스_카테고리 형태로 채워짐)
  5. 결과 확인 — Dashboard 진입 시 첫 카드가 자동 활성. 다른 카드 클릭으로 즉시 전환

3. 소스 추가 — 4가지 채널

3.1 Google Play

  1. Sources → 소스 유형 = Google Play
  2. 검색창에 앱 이름 입력 (예: spotify)
  3. 국가(country) 선택 — 리뷰 언어에 영향
  4. 후보 카드(아이콘 + 개발자명 + 평점) 중 원하는 항목 클릭
  5. 별칭·최대 개수 조정 → 추가

3.2 Apple App Store

절차는 Google Play와 동일. country가 더 중요 — Apple은 국가별 별도 리뷰 풀입니다. 미국(us)이 기본, 한국 리뷰가 필요하면 kr. 수집은 Apple 공식 RSS 엔드포인트(페이지당 50건, 최대 500건)를 사용합니다.

3.3 Reddit

사전 준비:

  1. reddit.com/prefs/apps에서 script 타입 앱 생성 (이메일 인증 안 됐으면 먼저 인증)
  2. 발급된 client_id + secret을 Render Environment에 등록:
    • REDDIT_CLIENT_ID
    • REDDIT_CLIENT_SECRET
    • REDDIT_USER_AGENT=web:review-collector:v0.1 (by /u/your_id)
  3. Render 재시작 후 Sources → Reddit 선택 → 서브레딧 검색 → 선택 → 추가

3.4 사용자 지정 URL

  1. Sources → 소스 유형 = Custom URL
  2. URL 목록 입력 (한 줄에 하나)
  3. CSS selector 옵션 (비워두면 readability로 자동 본문 추출)
  4. JS 렌더 필요 시 Playwright 옵션 (Render 무료 플랜에선 메모리 제약으로 비활성화 권장)

4. 카테고리 트리 만들기

  1. Categories → 루트 카테고리 추가
  2. 이름과 설명 입력 — 설명이 LLM 분류 기준
  3. 각 노드의 하위 추가로 자식 생성
설명 작성 팁: "결제 관련 (구독, 환불, 가격)" 같이 1~2줄로 키워드를 적어두면 LLM 분류 정확도가 크게 올라갑니다. 빈 설명은 분류 품질 저하의 가장 큰 원인.

5. 분석 실행 + 조사 카드 생성

분석은 단순한 "한 번 돌리고 끝"이 아니라 조사 카드를 만드는 과정입니다.

  1. Analyze 페이지 상단에 미분석/실패/전체 건수가 보임
  2. 모드: 미분석만 (보통) / 실패만 재시도 / 전체 재분석
  3. 분석 대상 소스: 체크박스로 분석할 소스 멀티 선택. 미체크 시 전체
  4. 분류 범위(루트 카테고리): 멀티 선택한 root의 subtree만 LLM에 노출되어 그 안에서 분류. 미체크 시 전체 트리
  5. 모델:
    모델속도·비용정확도추천
    Haiku매우 빠름·저렴좋음대량 분류, 초기 탐색
    Sonnet균형매우 좋음일반 분석
    Opus느리고 비쌈최고중요한 인사이트
  6. 요약 언어: 영어 / 한국어 / 자동
  7. 현재 조건에 해당하는 리뷰: 체크박스·모드 변경에 따라 분석 대상 건수가 실시간 표시됨
  8. 조사 카드로 저장:
    • 이름 칸이 자동으로 <선택한 소스들>_<선택한 카테고리들> 형태로 채워짐 (예: Cal AI Google Play, Cal AI App Store_사용성 UX/UI)
    • 그대로 두거나 직접 수정 가능. 비우면 카드는 만들지 않고 분석만 실행
  9. 분석 실행 → 진행률 표시 (2초 간격 폴링)
Anthropic 비용 주의: 분석 버튼은 누구나 누를 수 있으니 공개 URL이라면 Anthropic 콘솔에서 API 사용량을 정기적으로 모니터링하세요.

6. 대시보드 — 조사 카드 중심

6.1 첫 화면

대시보드 진입 시:

  • 저장된 카드가 1개 이상: 가장 최근 업데이트된 카드가 자동 활성되어 그 범위의 통계 표시
  • 카드가 0개: 빈 상태 메시지 표시 ("조사 카드가 없습니다. 분석 페이지에서 만드세요")

6.2 카드 그리드

상단에 모든 카드가 그리드로 표시됩니다. 각 카드:

  • 카드 이름 + 업데이트 날짜
  • 소스 chips (아이콘 포함) + 카테고리 chips
  • 활성 카드는 파란 테두리 + "활성" 뱃지
  • 우측 하단 삭제 버튼 — 카드 삭제 시 연결된 마인드맵도 함께 삭제됨 (CASCADE)

6.3 현재 보기 strip

카드 그리드 바로 아래에 읽기 전용으로 현재 활성 카드의 소스·카테고리가 표시됩니다 (수동 토글 불가).

6.4 패널들 (활성 카드 범위로 자동 필터)

  • 4개 요약 카드: 총 리뷰 / 평균 감성 / 평균 별점 / 최근 수집
  • 루트 카테고리별 감성 분포 (가로 누적 막대)
  • 소스별 분포 (도넛 + 표, 평균 별점 포함)
  • 카테고리별 누적 막대
  • 시간 추이 (주간/월간 × 5단계분포/평균점수)
  • 감성 원인 마인드맵 (다음 섹션 참고)
  • 최근 리뷰 미리보기

7. 감성 원인 마인드맵

활성 카드의 리뷰들을 LLM이 분석해서 "왜 이 감성이 나오는지" 테마를 추출합니다.

7.1 동작

  1. 대시보드에서 활성 카드 확인
  2. 마인드맵 패널의 감성 dropdown (매우 긍정 / 긍정 / 부정 / 매우 부정)
  3. 생성 클릭 → Claude가 최근 100건을 읽고 카테고리별로 그룹된 테마 5~8개 + 대표 인용구 반환
  4. 마인드맵에 3단계 트리: 감성 → 카테고리 → 테마(카운트) → 인용구
  5. 아래에 카테고리별 아코디언 (클릭으로 펼치기/접기) + 카운트 caveat 문구

7.2 인터랙션 (markmap.js 기반)

  • 노드 클릭 → 그 하위 가지 접기/펼치기
  • 드래그 → 전체 팬
  • 마우스 휠 → 확대/축소
  • 📐 Fit → 화면에 맞춤 (리셋)
  • ⬇️ SVG 다운로드 → 현재 화면 SVG로 저장

7.3 저장 (카드별)

  • 💾 저장 클릭 → 이름 입력 → 그 카드 소속 마인드맵으로 DB 저장
  • 저장된 마인드맵은 그 카드가 활성일 때만 dropdown에 표시 (다른 카드 활성 시엔 가려짐)
  • 카드 삭제 시 마인드맵도 CASCADE로 같이 삭제
  • SVG / 워크스페이스 export에 모두 포함됨
카운트 해석: 각 테마의 숫자는 그 테마를 주요 주제로 다룬 리뷰 수입니다. 한 리뷰는 한 번만 카운트되며, 카테고리 내 합산은 카테고리 총합 이하 (약간의 오차는 정상).

8. 리뷰 탐색 + 내보내기

  • Reviews 페이지 상단에도 조사 카드 그리드가 표시됨 — 클릭하면 그 카드 범위의 리뷰만 표시
  • 좌측 사이드바 필터(감성/기간/키워드)로 카드 범위 안에서 추가 필터링 가능
  • 우상단 CSV / JSON / Excel 내보내기 — 활성 카드 + 사이드바 필터가 그대로 반영된 결과만 다운로드
  • 헤더의 ✕ 필터 해제 → 카드 선택 없는 상태로 (전체 리뷰)

9. 워크스페이스 저장·불러오기

DB는 모든 방문자와 공유됩니다. 본인 조사를 따로 보관하려면 사이드바 워크스페이스 섹션:

  • 파일로 내보내기: 현재 상태(소스·카테고리·리뷰·분석·조사 카드·마인드맵 전부)를 JSON으로 다운로드
  • 파일 불러오기: 저장해뒀던 JSON 업로드 → 현재 DB를 그 시점 상태로 덮어씀
  • 새로 시작: 모든 데이터 비우고 처음부터

This app collects user reviews from multiple channels, classifies them with your own category tree, and grades each on a 5-band sentiment scale. The central organising primitive is the Investigation card: a saved (sources × categories) pair. Click one and every panel — dashboard charts, mind maps, the reviews list — scopes to it.

1. What this app does

Pull reviews from Google Play, Apple App Store, Reddit, and arbitrary web pages; classify each into your hierarchical category tree; grade each on a 5-band sentiment (very negative → very positive); then use Investigation cards to drill into any saved (sources × categories) subset.

What's an Investigation card? When you run analysis you bundle the chosen sources and root categories under a name. That bundle becomes a card on the dashboard and reviews pages. Click the card and every panel filters to that subset. Mind maps also live per card.

2. Quick start (5 steps)

  1. Add a source — register apps, subreddits, or URLs in Sources
  2. Define categories — build your classification tree in Categories
  3. Collect — click Start collection on each source card
  4. Create an Investigation + analyse — in Analyze, pick sources and root categories, give the run a name (auto-filled as sources_categories), and run
  5. Inspect — opening the Dashboard auto-activates the most recent card. Click a different card to switch context.

3. Add a source — 4 channels

3.1 Google Play

  1. Sources → source type = Google Play
  2. Search the app name (e.g. spotify)
  3. Pick a country — affects review language
  4. Click a candidate card → adjust label / max count → Add

3.2 Apple App Store

Same flow as Google Play. Country matters more here — Apple has a separate review pool per country store. We use Apple's official customer-reviews RSS (50 entries × 10 pages = max 500).

3.3 Reddit

  1. Create a script-type app at reddit.com/prefs/apps (verify email first if needed)
  2. Set Render env vars: REDDIT_CLIENT_ID, REDDIT_CLIENT_SECRET, REDDIT_USER_AGENT=web:review-collector:v0.1 (by /u/your_id)
  3. Sources → Reddit → search subreddit → pick → add

3.4 Custom URL

  1. Sources → source type = Custom URL
  2. One URL per line
  3. Optional CSS selectors (empty = readability auto-extract)
  4. Playwright toggle for JS-rendered pages (off by default on Render free for memory reasons)

4. Build a category tree

  1. Categories → Add root category
  2. Enter name + description — the description is the LLM's classification rubric
  3. Use each node's Add child to grow the tree
Description tip: A one-line description like "Billing-related (subscription, refund, pricing)" massively improves accuracy. Empty descriptions are the #1 cause of poor results.

5. Run analysis + create an Investigation

Analysis is no longer a one-shot — each run can also save a card.

  1. Header shows current unanalyzed / failed / total counts
  2. Mode: unanalyzed only (typical) / retry failed / re-analyze everything
  3. Source scope: multi-select checkboxes; unchecked = every source
  4. Category scope (roots): multi-select; LLM only sees those subtrees
  5. Model:
    ModelSpeed / CostAccuracyWhen
    HaikuFast / cheapGoodBulk, exploration
    SonnetBalancedVery goodGeneral use
    OpusSlower / pricierBestHigh-stakes insights
  6. Summary language: English / Korean / Auto
  7. Live count: a "Reviews matching the current selection: N" line updates as you toggle
  8. Save as investigation card:
    • Auto-filled with <sources>_<categories> the moment you tick boxes
    • Keep it or rewrite. Empty means "just run, don't save a card."
  9. Click Run analysis → polled progress bar.
Anthropic cost: the button is open to anyone with the URL — monitor usage in the Anthropic console.

6. Dashboard — driven by cards

6.1 First view

  • At least one card exists: the most recently updated card is auto-activated; panels render its scope.
  • No cards: empty-state placeholder pointing you to Analyze.

6.2 Cards grid

  • Each card shows label, update date, source chips with icons, category chips.
  • Active card highlighted with a blue border + "Active" badge.
  • Bottom-right delete button — deleting a card cascades to its saved mind maps.

6.3 "Currently viewing" strip

Read-only chips below the cards grid show the active sources and categories. No manual toggle — pick a different card to change scope.

6.4 Panels (all scoped to the active card)

  • 4 summary cards: total / avg sentiment / avg rating / last collected
  • Sentiment by root category (horizontal stacked bar)
  • By source (doughnut + table with avg rating)
  • By category (stacked bar)
  • Trend (weekly / monthly × 5-band / average)
  • Mind map (next section)
  • Recent reviews

7. Sentiment reasons mind map

The LLM reads up to 100 reviews of the chosen sentiment within the active card and extracts the dominant themes per category.

7.1 Workflow

  1. Pick a sentiment band from the dropdown (very_positive / positive / negative / very_negative)
  2. Click Generate → Claude returns 5–8 themes grouped by category, each with 2–3 representative quote snippets
  3. Mind map renders a 3-level tree: Sentiment → Category → Theme (count) → Quote
  4. Companion accordion below the mind map lets you click each category open / collapse

7.2 Interaction (markmap.js)

  • Click a node → fold / unfold its descendants
  • Drag → pan
  • Wheel → zoom
  • 📐 Fit → reset view
  • ⬇️ Download SVG → save the rendered map

7.3 Saving (per card)

  • 💾 Save → prompts for a label → persisted under the active card.
  • The "Saved mind maps" dropdown only lists this card's snapshots; switching cards swaps the list.
  • Deleting a card cascades to its saved mind maps.
  • Both SVG download and workspace export preserve them.
Counts: each theme's number is the count of reviews whose primary topic is that theme — a review is counted once. Per-category sums should be ≤ the category total; tiny drift is normal.

8. Reviews explorer + export

  • The Reviews page shows the same Investigation cards grid on top — click one to scope the table.
  • Sidebar filters (sentiment / dates / keyword) refine inside the active card.
  • Top-right CSV / JSON / Excel export — the active card + sidebar filters apply to the download.
  • "✕ Clear filter" header link drops the card filter back to "everything".

9. Workspace save / load

The DB is shared across visitors. To keep your investigation private, use the workspace block in the sidebar:

  • Export to file: full snapshot (sources, categories, reviews, analyses, investigations, mind maps) as JSON.
  • Import file: upload to overwrite the DB with that snapshot.
  • Start fresh: wipe everything.
