1. 이 앱이 하는 일

Google Play, Apple App Store, Reddit, 임의 웹사이트에서 사용자 리뷰를 모은 뒤, 본인이 정의한 계층형 카테고리로 자동 분류하고 각 리뷰의 5단계 감성(매우 부정 ~ 매우 긍정)을 평가합니다. 결과는 대시보드 차트와 리뷰 테이블에서 비교·필터링·내보내기 가능.

2. 빠른 시작 (5단계)

  1. 소스 추가 — 좌측 메뉴 Sources 에서 분석할 앱·서브레딧·URL 등록
  2. 카테고리 정의 — Categories 에서 분류 기준 트리 생성
  3. 수집 실행 — Sources 페이지에서 각 소스의 수집 시작 버튼 클릭
  4. 분석 실행 — Analyze 에서 모델 선택 후 실행
  5. 결과 확인 — Dashboard 차트와 Reviews 테이블

3. 소스 추가 — 4가지 채널

3.1 Google Play

  1. Sources → 소스 유형 = Google Play
  2. 검색창에 앱 이름 입력 (예: spotify)
  3. 국가(country) 선택 — 리뷰 언어에 영향
  4. 후보 카드(아이콘 + 개발자명 + 평점) 중 원하는 항목 클릭
  5. 별칭·최대 개수 조정 → 추가

3.2 Apple App Store

절차는 Google Play와 동일. 단 country가 더 중요 — Apple은 각 국가별로 별도 리뷰 풀입니다. 미국(us)이 기본, 한국 리뷰가 필요하면 kr.

3.3 Reddit

사전 준비가 필요합니다:

  1. reddit.com/prefs/apps 에서 script 타입 앱 생성 (이메일 인증 안 됐으면 먼저 인증)
  2. 발급된 client_id + secret을 Render Environment에 등록:
    • REDDIT_CLIENT_ID
    • REDDIT_CLIENT_SECRET
    • REDDIT_USER_AGENT=web:review-collector:v0.1 (by /u/your_id)
  3. Render가 자동 재시작된 후 Sources → Reddit 선택
  4. 서브레딧 이름 검색 (예: spotify) → 후보 클릭
  5. 옵션 조정:
    • 검색어: 비워두면 서브레딧 전체
    • 정렬: new(최신) / top(인기) / hot / relevance
    • 기간: top/relevance일 때 적용
    • 댓글 포함: 게시글의 상위 N개 댓글도 수집
  6. 추가 클릭

3.4 사용자 지정 URL

  1. Sources → 소스 유형 = Custom URL
  2. URL 목록 입력 (한 줄에 하나)
  3. CSS selector 입력 (선택):
    • item_selector: 반복되는 리뷰 블록 (예: .review-item)
    • text_selector, author_selector, date_selector, rating_selector: 각 블록 안에서 추출할 요소
    • 비워두면 readability로 본문 자동 추출
  4. JavaScript 렌더가 필요한 페이지면 Playwright 사용 체크 (현재 Render 무료 플랜에선 메모리 제약으로 비활성화 권장)

4. 카테고리 트리 만들기

  1. Categories 페이지 → 우상단 루트 카테고리 추가
  2. 이름과 설명 입력 — 설명이 LLM의 분류 기준이 됩니다
  3. 각 노드의 하위 추가로 자식 생성
설명 작성 팁: "결제 관련 (구독, 환불, 가격)" 같이 1~2줄로 키워드를 적어두면 LLM이 훨씬 정확하게 분류합니다. 빈 설명은 분류 품질 저하의 가장 큰 원인.

깊이 제한은 없지만 실용상 3단계 이내 권장. 예시:

사용성 UX/UI
├─ 온보딩          (가입·튜토리얼·첫 사용 경험)
├─ 핵심 UX         (메인 화면·네비게이션)
└─ 결제·구독        (구독 가입·해지·환불·가격)
성능
├─ 속도            (로딩·반응 속도)
└─ 크래시·버그     (앱 종료·오류 메시지)
기타                (위에 해당하지 않는 모든 리뷰)

5. 분석 실행

  1. Analyze 페이지 접속 — 상단에 미분석/실패/전체 건수 표시
  2. 모드 선택:
    • 미분석만: 아직 분석 안 된 리뷰만 처리 (보통 사용)
    • 실패만 재시도: 이전에 실패한 분석만 다시
    • 전체 재분석: 모든 리뷰 재분석 (카테고리 트리 바꾼 후)
  3. 분류 범위 (루트 카테고리) 선택 (선택사항):
    • 특정 root 체크 → LLM에 그 subtree만 노출되어 그 안에서 분류
    • 아무것도 안 체크 → 전체 트리 사용
  4. 모델 선택:
    모델속도·비용정확도추천 상황
    Haiku매우 빠름·저렴좋음대량 분류, 초기 탐색
    Sonnet균형매우 좋음일반 분석
    Opus느리고 비쌈최고중요한 인사이트 도출
  5. 요약 언어: 영어 / 한국어 / 자동(리뷰 언어 따라)
  6. 분석 실행 → 진행률 바 표시 (2초 간격 폴링)
주의: 분석은 Anthropic API를 호출하므로 비용이 발생합니다. 본 사이트는 인증 없이 공개돼 있어 누구나 분석 버튼을 누를 수 있으니, Render Dashboard에서 사용량 모니터링을 권장합니다.

6. 대시보드 활용

6.1 상단 필터 카드

  • 소스 칩: 멀티 선택. 선택한 소스만 모든 차트에 반영. 미선택 = 전체
  • 카테고리 칩: 루트 카테고리 멀티 선택. 그 root의 모든 후손이 통계에 포함
  • 둘 다 선택하면 AND로 결합 (예: "Cal AI App Store" + "UX" → 그 앱의 UX 분석만)

6.2 4개 카드

  • 총 리뷰: 필터 적용 후 리뷰 수
  • 평균 감성 점수: 1~5 평균 (1=매우 부정, 5=매우 긍정)
  • 평균 별점: Google Play / App Store만 별점 보유
  • 최근 수집: 가장 최근 수집된 리뷰 시각

6.3 차트

  • 루트 카테고리별 감성 분포: 가로 누적 막대. 카테고리 1개 = 막대 1개
  • 출처별 분포: 도넛 + 표 (소스 타입·평점도 함께)
  • 카테고리별 누적 막대: 모든 leaf 카테고리의 5단계 감성
  • 시간 추이: 주간/월간 × 5단계 분포/평균 점수 4가지 모드 전환 가능

6.4 Reviews 탐색

좌측 필터(소스/카테고리/감성/기간/키워드) → 페이지네이션 → 우상단 내보내기 CSV/JSON/Excel

7. 워크스페이스 저장·불러오기

이 사이트의 DB는 모든 방문자와 공유됩니다. 본인 조사를 따로 보관하려면:

  • 📤 파일로 내보내기: 현재 상태(소스·카테고리·리뷰·분석 전부)를 JSON 파일로 다운로드
  • 📥 파일 불러오기: 저장해뒀던 JSON 업로드 → 현재 DB를 그 시점 상태로 덮어씀
  • 🗑 새로 시작: 모든 데이터 비우고 처음부터

사이드바 하단의 워크스페이스 섹션에서 이용하세요.

This is a single-user web app for collecting reviews from multiple channels, classifying them with a user-defined category tree, and grading each on a 5-band sentiment scale. Follow the steps below; you can reach your first analysis in about 10 minutes.

1. What this app does

Pull reviews from Google Play, Apple App Store, Reddit, and arbitrary web pages, then automatically classify each into a hierarchical category tree you define and grade it on a 5-band sentiment scale (very negative → very positive). Compare, filter, and export the results in the dashboard and reviews tables.

2. Quick start (5 steps)

  1. Add a source — register the app, subreddit, or URL in Sources
  2. Define categories — build your classification tree in Categories
  3. Collect — click Start collection on each source card
  4. Analyze — pick a model in Analyze and run
  5. Inspect — charts on the Dashboard, rows in Reviews

3. Add a source — 4 channels

3.1 Google Play

  1. Sources → source type = Google Play
  2. Type the app name in the search box (e.g. spotify)
  3. Pick a country — affects review language
  4. Click the candidate card (icon + developer + rating)
  5. Adjust label / max count → Add

3.2 Apple App Store

Same flow as Google Play. Country matters more here — Apple has a separate review pool per country store. Default us; pick kr for Korean reviews, etc.

3.3 Reddit

One-time setup required:

  1. Create a script-type app at reddit.com/prefs/apps (verify your email first if you haven't)
  2. Set the resulting credentials in Render Environment:
    • REDDIT_CLIENT_ID
    • REDDIT_CLIENT_SECRET
    • REDDIT_USER_AGENT=web:review-collector:v0.1 (by /u/your_id)
  3. After Render restarts, Sources → Reddit
  4. Search by subreddit name (e.g. spotify) → click candidate
  5. Adjust options:
    • Search query: empty = the whole subreddit
    • Sort: new / top / hot / relevance
    • Time filter: applies to top/relevance
    • Include comments: also fetch top N comments per submission
  6. Click Add

3.4 Custom URL

  1. Sources → source type = Custom URL
  2. Enter URLs, one per line
  3. CSS selectors (optional):
    • item_selector: the repeated review block (e.g. .review-item)
    • text_selector, author_selector, date_selector, rating_selector: extracted within each block
    • Leave blank → falls back to readability auto-extraction
  4. Check Use Playwright for pages requiring JS rendering (disabled on Render free tier by default to avoid OOM)

4. Build a category tree

  1. Categories → Add root category (top-right)
  2. Enter name + description — the description is the LLM's classification rubric
  3. Use each node's Add child to grow the tree
Description tip: A one-line description like "Billing-related (subscription, refund, pricing)" massively improves classification accuracy. Empty descriptions are the #1 cause of poor results.

No depth limit, but 3 levels is recommended in practice. Example:

UX
├─ Onboarding       (signup, tutorial, first-use experience)
├─ Core UX          (main screen, navigation)
└─ Billing          (subscription, cancellation, refund, pricing)
Performance
├─ Speed            (loading, responsiveness)
└─ Crashes & bugs   (app exits, error messages)
Other               (anything not fitting the above)

5. Run analysis

  1. Analyze page shows current unanalyzed / failed / total counts
  2. Pick a mode:
    • Analyze unanalyzed only: typical use
    • Retry failed only: re-run previous failures
    • Re-analyze everything: useful after editing the category tree
  3. Category scope (root) — optional:
    • Check one or more roots → LLM only sees those subtrees
    • Leave all unchecked → full tree
  4. Model:
    ModelSpeed / CostAccuracyWhen to use
    HaikuFast / cheapGoodBulk classification, early exploration
    SonnetBalancedVery goodGeneral analysis
    OpusSlower / pricierBestImportant insights
  5. Summary language: English / Korean / Auto (match review)
  6. Click Run analysis → progress bar (polls every 2 s)
Heads up: Analysis hits the Anthropic API, which costs money. This site has no auth, so anyone with the URL can press the button. Monitor usage in the Anthropic console.

6. Reading the dashboard

6.1 Top filter card

  • Source chips: multi-select; only selected sources contribute. Empty = all
  • Category chips: multi-select root categories; all descendants are included
  • Combining both is an AND (e.g. "Cal AI App Store" + "UX" → only UX analyses for that app)

6.2 Four summary cards

  • Total reviews after filters
  • Average sentiment on the 1–5 scale
  • Average rating (Google Play / App Store only)
  • Last collected timestamp

6.3 Charts

  • Sentiment by root category: horizontal stacked bar
  • By source: doughnut + table (source type + rating shown)
  • By category: stacked bar for every leaf
  • Trend: weekly/monthly × 5-band/average — toggle to compare

6.4 Reviews explorer

Left filter pane (source/category/sentiment/date/keyword) → pagination → Export CSV/JSON/Excel top-right.

7. Workspace save / load

The database is shared across every visitor. To keep your investigation separate:

  • 📤 Export to file: download the current state (sources, categories, reviews, analyses) as JSON
  • 📥 Import file: upload a saved JSON → overwrites the current DB with that snapshot
  • 🗑 Start fresh: wipe everything and begin from zero

The controls live in the sidebar workspace section.
